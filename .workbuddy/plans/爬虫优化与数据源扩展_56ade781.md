## 产品需求

优化心理学内容爬虫系统，实现增量数据更新和去重机制，丰富数据来源。

## 核心功能

1. **内容指纹去重系统**

- 使用内容哈希(SHA-256)作为数据唯一标识
- 爬取前先检查指纹是否已存在
- 支持多种去重策略：指纹去重 + URL去重 + 标题去重

2. **数据源扩展**

- 扩展英文权威来源（增加更多心理学主题分类）
- 增加中文优质心理学网站
- 增加国际期刊/研究机构来源

3. **爬取历史管理**

- 记录已爬取URL列表
- 记录成功/失败状态
- 支持增量爬取（只爬取新内容）

4. **增量更新优化**

- 与现有KnowledgeManager集成
- 统一的去重检查流程
- 详细的更新日志和统计

## 预期效果

- 避免重复插入已存在数据
- 数据源覆盖率提升3倍以上
- 爬取效率提升（跳过已爬URL）
- 数据质量提升（更多权威来源）

## 技术方案

### 1. 技术选型

- **Node.js原生模块**: crypto (内置SHA-256哈希)
- **HTTP客户端**: 原生http/https（复用现有代码）
- **数据存储**: Upstash Search（复用现有配置）
- **指纹算法**: SHA-256哈希，截取前16位作为短指纹

### 2. 实现架构

#### 2.1 指纹生成器 (FingerprintGenerator)

```javascript
// 生成内容指纹
function generateFingerprint(content) {
  const hash = crypto.createHash('sha256');
  hash.update(content.normalize('NFC'));
  return hash.digest('hex').substring(0, 16);
}

// 多重去重键
{
  fingerprint: "a1b2c3d4e5f67890",  // 内容指纹
  urlHash: "xxx",                    // URL哈希
  titleHash: "xxx",                  // 标题哈希
  sourceUrl: "https://...",          // 来源URL
  title: "..."                       // 标题
}
```

#### 2.2 爬取历史管理器 (CrawlHistoryManager)

- 存储结构: Map<url, {status, fingerprint, timestamp}>
- 持久化: 存储到Upstash metadata索引
- 检查逻辑: URL已爬 且 指纹未变 → 跳过

```

数据存储 → 历史记录更新
```

### 5. 关键实现细节

#### 去重检查优先级

1. URL已爬 + 指纹匹配 → 跳过
2. URL已爬 + 指纹不匹配 → 更新内容
3. URL未爬 → 新增
