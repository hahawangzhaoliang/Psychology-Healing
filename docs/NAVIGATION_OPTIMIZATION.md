# 导航结构优化方案

## 📊 当前问题分析

### 现有页面（11个）
1. `index.html` - 首页
2. `practice.html` - 疗愈练习
3. `emotion.html` - 情绪觉察
4. `knowledge-graph.html` - 知识图谱
5. `about.html` - 关于我们
6. `article.html` - 疗愈内容
7. `charging-station.html` - 5分钟充电站
8. `community.html` - 微光社区
9. `relax.html` - 放松练习
10. `role.html` - 角色探索
11. `test.html` - 重定向到emotion.html

### 主要问题

1. **功能重复**
   - `practice.html` 和 `relax.html` 都是练习相关
   - 用户不清楚该去哪个页面

2. **导航层级不清晰**
   - 主导航只有5项，但实际有11个页面
   - 重要功能（充电站、社区、角色探索）不在主导航

3. **用户流程不顺畅**
   - 缺少明确的使用路径引导
   - 新用户不知道从哪里开始

4. **命名不一致**
   - "疗愈练习" vs "放松练习" vs "疗愈内容"
   - 概念混淆

---

## 🎯 优化方案

### 方案A：简化导航（推荐）

**主导航（5项）**：
```
首页 → 疗愈练习 → 情绪觉察 → 知识学习 → 关于我们
```

**详细结构**：

1. **首页** (`index.html`)
   - 核心价值展示
   - 快速入口卡片
   - 功能推荐

2. **疗愈练习** (`practice.html`) ⭐ 合并页面
   - 四种核心练习（呼吸、身体扫描、冥想、放松）
   - 5分钟充电站（作为子功能或快速入口）
   - 练习记录和统计

3. **情绪觉察** (`emotion.html`)
   - Plutchik情绪轮盘
   - 情绪温度计
   - 情绪曲线
   - 调节建议

4. **知识学习** (`knowledge.html`) ⭐ 新建整合页
   - 知识图谱（可视化）
   - 疗愈文章（列表）
   - 学习路径推荐

5. **关于我们** (`about.html`)
   - 平台介绍
   - 科学依据
   - 联系方式

**次要功能（通过首页或子页面访问）**：
- 微光社区 (`community.html`) - 从首页或情绪觉察页进入
- 角色探索 (`role.html`) - 从首页或情绪觉察页进入
- 5分钟充电站 (`charging-station.html`) - 从疗愈练习页进入

---

### 方案B：扩展导航

**主导航（7项）**：
```
首页 → 疗愈练习 → 情绪觉察 → 知识图谱 → 微光社区 → 角色探索 → 关于我们
```

**优点**：所有功能都在主导航
**缺点**：导航项过多，移动端体验不佳

---

## 🎨 推荐方案：方案A + 用户流程优化

### 主导航结构

```html
<nav>
  <a href="index.html">首页</a>
  <a href="practice.html">疗愈练习</a>
  <a href="emotion.html">情绪觉察</a>
  <a href="knowledge.html">知识学习</a>
  <a href="about.html">关于我们</a>
</nav>
```

### 用户使用流程

**新用户路径**：
```
首页 → 了解平台 → 快速体验（1分钟呼吸）→ 情绪觉察 → 获得建议 → 深度练习
```

**日常使用路径**：
```
情绪觉察（记录当前状态）→ 疗愈练习（针对性练习）→ 知识学习（了解更多）
```

**深度用户路径**：
```
知识图谱（系统学习）→ 角色探索（自我认知）→ 微光社区（分享交流）
```

## 📝 具体实施步骤

### 第1步：合并重复页面

**合并 practice.html 和 relax.html**：
- 保留 `practice.html` 作为主页面
- 将 `relax.html` 的内容整合到 `practice.html`
- `relax.html` 改为重定向到 `practice.html`

### 第2步：创建知识学习页面

**新建 `knowledge.html`**：
- 整合知识图谱和疗愈文章
- 提供两种学习模式：可视化图谱 + 文章列表
- 保留 `knowledge-graph.html` 和 `article.html` 作为子页面

### 第3步：优化首页卡片

**首页功能卡片**：
1. 核心功能（4个）：
   - 疗愈练习
   - 情绪觉察
   - 角色探索
   - 知识学习

2. 特色功能（3个）：
   - 5分钟充电站
   - 情绪温度计
   - 微光社区

### 第4步：统一导航更新

**所有页面导航统一为**：
- 首页
- 疗愈练习
- 情绪觉察
- 知识学习
- 关于我们

## 💡 优化后的优势

1. **导航清晰**：5个主导航项，层次分明
2. **流程顺畅**：明确的使用路径引导
3. **功能聚焦**：每个导航项功能明确，不重复
4. **易于扩展**：次要功能通过子页面或首页入口访问
5. **移动端友好**：导航项数量适中
